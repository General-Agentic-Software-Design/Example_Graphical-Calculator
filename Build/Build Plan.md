4. Ensure all requirements specified in the user stories are fully implemented.  
5. Include:

   - Detailed test cases  
   - Acceptance tests  
6. Place all generated GASD files under the **Design** folder.  
7. After completion, request human review before proceeding to the next step.
8. Once human review is completed and approved, proceed to the implementation step.
9. **CRITICAL**: All generated GASD design files MUST strictly use the official keywords from the **gasd-1.0.0.gasd** specification only. Non-standard keywords are strictly prohibited.
10. **MANDATORY**: The final implementation MUST be **100% in compliance with the GASD design** specified in the Design/ folder.

1. Read and follow **Design/gasd files** faithfully until fully proficient.  
2. Use this expertise to generate complete implementation files for the calculator.  
3. **MANDATORY**: The target platform MUST be either **React (JavaScript/TypeScript)** or **Java (Swing/AWT)**.
4. **DECISION GATE**: The chosen target platform MUST be presented to and confirmed by the user before any code generation begins.
5. Ensure all requirements specified in gasd files are fully implemented for the confirmed target.  

6. Place all generated implementation files under target-specific subfolders in the **Implementation** folder (e.g., `Implementation/React/` or `Implementation/Java/`).  
7. After completion, request human review before proceeding to the next step.
